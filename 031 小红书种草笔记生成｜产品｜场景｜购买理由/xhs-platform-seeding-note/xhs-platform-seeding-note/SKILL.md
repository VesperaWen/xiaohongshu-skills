# 小红书种草笔记生成｜产品｜场景｜购买理由

## Overview

使用这个 Skill 处理「小红书种草笔记生成｜产品｜场景｜购买理由」相关小红书运营任务。它属于「图文笔记创作」，重点解决：输入产品和卖点，生成自然的小红书种草笔记。

## Workflow

1. 明确任务目标：爆款拆解、选题、笔记创作、视频脚本、封面设计、账号增长、电商转化、达人投放、合规审查或总诊断。
2. 收集上下文：账号定位、目标用户、赛道、商品/服务、参考笔记、历史数据、素材现状和用户卡点。
3. 按小红书链路拆解：选题是否有需求，封面标题是否有点击，正文是否有收藏/评论点，转化承接是否自然。
4. 输出结构化结果：给出笔记正文、可复制文案、执行步骤、检查清单和下一轮测试。
5. 做合规与原创检查：避免搬运、夸大、虚假体验、违规导流、未经授权素材和敏感功效表达。
6. 给复盘指标：明确观察曝光、点击、阅读完成、点赞、收藏、评论、关注、私信、商品点击或成交。

## Xiaohongshu Patterns

- 把结论拆成：观察事实、判断假设、可执行动作，不要把猜测写成定论。
- 优先输出可直接复制、可直接执行、可复盘的内容。
- 每个方案都要说明适合什么账号阶段、什么赛道、什么内容形式。
- 涉及爆款学习时，只迁移结构、用户洞察和表达方法，不复制原文和素材。
- 文案要像真实分享，避免硬广腔和过度AI味。
- 标题、开头、正文、标签、结尾要形成一条完整阅读路径。

## Output Format

- 任务诊断：一句话说明当前最关键问题。
- 核心方案：3-5 个优先动作，每个动作含目的、做法、素材和指标。
- 可复制内容：按任务输出标题、正文、脚本、评论回复、brief、表格或审核意见。
- 风险提醒：列出可能的原创、版权、平台规则、功效宣传和账号安全风险。
- 下一步：今天可做、本周可测、下次复盘要看的指标。

## 通用边界

- 先确认账号类型、赛道、目标用户、内容形式、商业目标、已有数据和当前卡点；信息不足时先写明假设。
- 不直接搬运爆款内容，不伪造真实体验，不编造数据、资质、用户评价或合作背书。
- 不提供刷量、刷赞、刷评、批量骚扰、规避平台风控、绕过审核、诱导违规导流等方案。
- 涉及医疗、保健、护肤功效、母婴、金融、二奢、减肥、法律等敏感方向时，先做资质和宣传风险检查。
- 自动发布、采集、评论、点赞、私信工具只能作为风险审查对象；默认输出人工确认的运营辅助流程。
- 平台规则、投放规则、类目要求和审核尺度会变化；关键发布、投放或店铺动作前，应查询当前官方规则或让用户提供后台要求。
